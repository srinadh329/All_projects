Essential features

User authentication
The bot authenticates the user by generating a verification code associated with the user and stores it. User authentication is performed to ensure, the user who has authorised the bot with the identity user is the same user chatting with the bot.
Pre-made industry-specific modules
The chatbot is pre-trained to understand brand/industry specific knowledge and terms. Even better, it’s pre-configured to resolve common customer requests of particular industry modules, such as banking, insurance, telecom and more.
Multi-agent support
Multi-agent allows customers to connect with a Human.
Proactive abilities
In order to be effective, our chatbot has the capability to do more than simply wait for someone to ask it a question. With a touch of pro-activeness and well-structured scenarios, it is capable of asking user questions, initiating a dialogue and even awaking dormant prospects providing them with a personalised experience.
Seamless transition to a live agent
Conversations in a virtual agent are easily transferred to a live agent. This transfer happens automatically only when an unrecoverable error occurs, which can also be triggered via a script in a topic, or when manually selected by the user. The conversation is automatically routed to the appropriate chat support queue available in the connect support.
General intents included
Intents are intentions of the end-user. These intentions or intents are conveyed by the user to your bot. Having general affirmative and negative intents help you handle all other intents like “Ok”, “Yes please”, “Nope” and rather take them in context with the conversation, bot had with the client.
Training message templates
These allow you to override the default message formatting using markdown and apply templates, in order to display custom formatted bot messages to users. Message templates authorises you to insert a field in your message using a placeholder, which will be replaced with the corresponding value.
